# Booter 智能控制与伴侣系统使用指南

Booter 是一套现代化、高安全性、基于端对端控制的智能设备唤醒与关机管理系统。通过集成米家生态（智能插座唤醒）与自定义的伴侣程序（QUIC 强验证关机控制），它能够完美解决虚拟机环境中无法控制物理宿主机电源的痛点。

---

## 🛠️ 1. 系统架构概览

整个系统由三大核心组件构成：

1. **Booter Server (中央服务端)**: 系统的核心大脑，负责协调前端命令、处理权限、通过米家 API 唤醒设备，以及通过 QUIC 协议管理所有连接的伴侣。
2. **Booter Web (前端仪表盘)**: 采用现代 Web 框架构建，通过 WebTransport 与服务端建立高速、低延迟的双向通讯流，提供图形化操作与配置界面。
3. **Booter Companion (伴侣程序)**: 部署在受控宿主机（如 PVE 宿主机）上的极简无头客户端。它通过 QUIC 协议与服务端保持长连接，静默等待并执行服务端的指令（如关机）。

---

## 🚀 2. 部署与启动

### 2.1 准备环境与证书
服务端要求必须配置有效的 HTTPS 证书（因为 WebTransport 和 QUIC 均强制要求 TLS 1.3）：
```powershell
# 在服务端目录下生成自签证书
cd booter-server
./generate_cert.ps1
```
请确保 `booter-server/booter.toml` 配置文件中正确指向了生成的 `cert.pem` 和 `key.pem`。

### 2.2 启动服务端
服务端使用 Rust 构建，启动时会自动初始化内嵌的 SQLite 数据库：
```powershell
cd booter-server
cargo run --release
```
- Web HTTP 服务与 WebTransport 会绑定在 `0.0.0.0:8080`。
- 伴侣连接的 Raw QUIC 服务会绑定在 `0.0.0.0:8081`。

> [!TIP]
> **首次运行如何配置管理员？**
> 执行 `cargo run -- reset-admin` 即可在控制台生成管理员专属的 **TOTP 密钥与二维码**。将其扫码加入您的 Google Authenticator 或其他验证器中，即可使用动态密码登录前端 Admin 面板。

### 2.3 启动前端界面
如果您在开发环境下，可以单独运行前端（生产环境中服务端已静态内嵌打包后的前端文件）：
```powershell
cd booter-web
npm install
npm run dev
```

## 💻 3. 前端操作与配置指南

浏览器访问 `https://127.0.0.1:8080`（请在浏览器设置或快捷方式中开启相应的 flag 以信任自签证书，或输入 `thisisunsafe`）。

### 3.1 登录与权限划分
- **管理员登录 (`/admin/login`)**: 使用 `admin@booter` 配合刚才生成的 6 位动态密码登入，登入后可进入“管理面板”。
- **普通用户登录 (`/login`)**: 必须由管理员在“管理面板”的白名单中添加邮箱后，通过请求获取基于邮件的 一次性验证码 (OTP) 登录，只能进行开关机操作，无权修改系统配置。

### 3.2 管理面板核心功能 (`/admin`)

管理面板是您统筹一切的核心，包含三大功能区：

#### A. 米家配置 (用于物理开机)
1. 点击“生成登录二维码”，使用米家 APP 扫码授权。
2. 授权成功后，下拉菜单会自动列出您账号下的智能设备。
3. 选择控制开机的**智能插座**。后续当您在主控制台点击“唤醒”时，系统将调用米家 API 开启该插座电源。

#### B. 规则引擎 Rhai (用于关机安全拦截)
您可以编写一段类似 Rust 语法的拦截脚本。当用户点击“关机”时，服务端会先执行此脚本：
```rust
fn should_allow_shutdown() {
    // 比如：判断某服务是否在运行，或只允许在凌晨关机
    return true; // 返回 true 允许下发指令，false 则拦截
}
```

#### C. 伴侣管理中心 (用于配置受控终端)
1. 点击【+ 新增伴侣】。
2. 列表中将为您生成一个独一无二的 **UUID**（这就是该伴侣的合法身份证）。
3. 点击【编辑】可自定义名称和探针（如果该宿主只负责关机不需要探针，保留空的 `{}` 即可）。

---

## 🤖 4. 在宿主机上部署伴侣 (Companion)

获取到了 UUID 后，您就可以在需要受控的机器（比如 PVE 宿主机）上运行伴侣程序了。

启动伴侣程序时，通过环境变量注入目标服务器和刚生成的 UUID：

```powershell
# Windows 环境
$env:BOOTER_SERVER_HOST="您的服务端IP"
$env:BOOTER_CLIENT_ID="面板上生成的UUID"
# 如果服务端是自签证书，需要开启此参数跳过严格校验
$env:BOOTER_ALLOW_INSECURE="true"

cd booter-companion
cargo run --release
```

**连接机制说明：**
伴侣启动后将通过 UDP 8081 端口发起到服务端的 QUIC 连接。服务端核实 UUID 存在后立刻放行，并将其隐匿在后台。此时，在前端控制台即可看到“节点在线”的状态亮起。

---

## 🎮 5. 最终效果展示

一切配置就绪后，回到前台的 **Dashboard 控制台**：

- **唤醒电脑**：点击后，服务端悄无声息地向米家服务器发送指令，您的物理智能插座通电，宿主机被拉起。
- **状态监控**：一旦宿主机内的虚拟机启动并拉起了服务端，伴侣程序也会随后连上，页面状态灯瞬间从暗红转为**翠绿（节点在线）**。
- **关机执行**：点击“在伴侣程序上执行规则”，只要 Rhai 脚本通过校验，该指令便会如同闪电般通过 QUIC 管道下发至伴侣。伴侣程序随即在宿主机层面执行真正的 `shutdown` 物理关机。

至此，您已经完全掌控了跨越虚拟机和物理机的电源管理闭环！
